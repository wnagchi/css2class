# 我用Tailwind写小程序，结果被rpx折磨到怀疑人生...直到遇见它

## 那个让我崩溃的下午

还记得那个周五下午，我兴冲冲地想把Tailwind CSS引入小程序项目，想着终于可以告别那些重复的CSS代码了。结果呢？**我花了整整3个小时在配置rpx单位转换上，最后还失败了。**

那一刻，我内心是崩溃的。Tailwind确实很香，但它在小程序面前就像个"水土不服"的外国人——明明很强大，但就是不懂rpx这个"方言"。

## 直到我发现了Class2CSS

说实话，第一次看到Class2CSS的时候，我内心是怀疑的："又一个想蹭Tailwind热度的工具？"但用了一周后，我只想说：**真香！**

这不是一个"山寨版Tailwind"，而是一个真正**为中国小程序开发者而生**的工具。

## 为什么说它比Tailwind更适合小程序？

### 1. rpx？开箱即用，不需要你操心

用Tailwind的时候，每次写样式我都要想："这个值应该转成多少rpx？"然后去查转换比例，再手动计算。**累不累？累死了！**

Class2CSS呢？你写 `class="m-10"`，它自动给你生成 `margin: 20rpx`（根据你的转换比例）。你写 `class="w-100px"`，它知道你要的是px，不会乱转。**就像有个懂你的助手，你写什么它都懂。**

```html
<!-- 就这么简单，不需要想单位 -->
<view class="m-10 w-100 h-200">
  内容
</view>
```

### 2. 配置？你想怎么来就怎么来

Tailwind的配置就像"别人家的房子"——你可以装修，但框架是固定的。Class2CSS呢？**这是你自己的房子，想怎么设计就怎么设计。**

你想用 `m` 表示margin？可以。你想用 `spacing` 表示margin？也可以。你想用 `flex-center` 表示居中？随便你。**完全由你说了算。**

```javascript
// 这就是你的"设计图纸"，想怎么改就怎么改
cssName: {
  "m": { classArr: ["margin"], unit: "rpx" },
  "我的间距": { classArr: ["margin"], unit: "rpx" },  // 甚至可以用中文！
}
```

### 3. 性能？90%+的缓存命中率，快到飞起

第一次用的时候，我还在想："这扫描文件会不会很慢？"结果发现，**90%+的缓存命中率**，文件变更时几乎秒级响应。而且内存占用还减少了40%。

Tailwind的JIT模式虽然快，但需要构建工具配合。Class2CSS呢？**直接运行，不需要额外配置。**

### 4. 学习成本？5分钟上手

Tailwind的文档虽然详细，但你要记住那些类名规则：`mt-4`、`mb-6`、`px-8`...用Class2CSS呢？**你定义什么规则，就用什么规则。** 团队内部统一一下，大家都能快速上手。

## 真实对比：我用Tailwind vs Class2CSS的一天

### 上午10点：开始写一个卡片组件

**用Tailwind：**
1. 查文档，确认rpx转换比例
2. 配置tailwind.config.js
3. 写样式，手动计算rpx值
4. 发现某些样式不支持，需要扩展配置
5. **耗时：30分钟**

**用Class2CSS：**
1. 写 `class="p-20 m-10 rounded-10"`
2. 保存，CSS自动生成
3. **耗时：2分钟**

### 下午2点：需要修改设计规范

**用Tailwind：**
1. 修改tailwind.config.js
2. 重新构建
3. 检查是否有遗漏
4. **耗时：15分钟**

**用Class2CSS：**
1. 修改配置文件
2. 自动热更新
3. **耗时：10秒**

## 那些让我"哇塞"的瞬间

### 智能单位检测，真的智能

```html
<!-- 你写什么单位，它就用什么单位 -->
<view class="w-100px">      <!-- 它知道你要px -->
<view class="w-100">        <!-- 它知道你要rpx -->
<view class="opacity-50">   <!-- 它知道这个不需要单位 -->
```

**就像有个懂你的同事，不需要解释。**

### 配置诊断，自动帮你找问题

有时候配置写错了，Class2CSS会直接告诉你："这里有问题，建议这样改。"**不用自己debug到怀疑人生。**

### 增量更新，只改你改的

文件改了100个，但只改了1个类名？Class2CSS只更新那1个，不会全量重建。**这就是效率。**

## 什么时候选Class2CSS？什么时候选Tailwind？

### 选Class2CSS，如果你：

- ✅ **在做小程序**：rpx支持是刚需
- ✅ **想要完全控制**：不想被预设规则束缚
- ✅ **团队有统一规范**：可以自定义一套标准
- ✅ **追求极致性能**：需要精确控制CSS大小
- ✅ **不想学新语法**：用自己熟悉的规则

### 选Tailwind，如果你：

- ⚠️ **在做Web项目**：标准Web开发
- ⚠️ **需要快速原型**：Tailwind的预设样式确实丰富
- ⚠️ **团队已经熟悉**：大家都会用Tailwind
- ⚠️ **需要生态支持**：Tailwind的插件和工具确实多
